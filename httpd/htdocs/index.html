Hello from Apache httpd running in Docker. To create a reverse proxy:
  1. enable mod_proxy by removing the # comment sign from the following lines:
    • LoadModule proxy_module modules/mod_proxy.so
    • LoadModule proxy_http_module modules/mod_proxy_http.so
    • LoadModule ssl_module modules/mod_ssl.so
  2. define a virtual host, either directly in the configuration file or preferably inside a virtual host: 
    SSLProxyEngine on  # since we are proxy-ing to an endpoint with an SSL certificate
ProxyPass /api/ https://ckan-dataplatform-nl.dataplatform.nl/api/
ProxyPassReverse /api/ https://ckan-dataplatform-nl.dataplatform.nl/api/
  3. You can now do an API call via relative path, without CORS issues
